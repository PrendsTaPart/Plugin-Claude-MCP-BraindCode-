# Patch d'enrichissement — `rapido-forge:scale-blue-ocean`

> **Cible** : plugin `rapido-forge`, skill `scale-blue-ocean`.
> **Idées** : W. Chan Kim & Renée Mauborgne, *Blue Ocean Strategy* (2005).
> **Distillation source** : founder-playbook (MIT © 2026 AgentSeal). Reformulation
> FR, aucun extrait. **Ne pas modifier le skill dans ce document.**

Le skill couvre déjà ERRC + Strategy Canvas + Value Innovation. **Frameworks
manquants** :

## 1. Six Paths (reconstruire les frontières)
Regarder au-delà de son secteur : industries alternatives, groupes stratégiques,
chaîne d'acheteurs, produits complémentaires, appel fonctionnel↔émotionnel,
tendances. *Cas Rapido* : trouver un angle d'offre inédit pour un client
FoodEatUp en regardant les complémentaires (livraison, événementiel).

## 2. Les 3 cercles de non-clients
« Bientôt » (au bord du départ) · « refusants » · « inexplorés ». Viser les
**non-clients avant les clients** pour agrandir le marché.

## 3. Buyer Utility Map
6 étapes du cycle d'usage × 6 leviers d'utilité = 36 cases pour repérer les
blocages d'utilité non résolus par le secteur.

## 4. Séquence stratégique (test avant lancement)
**Utilité → Prix (accessible à la masse) → Coût (atteignable) → Adoption
(obstacles)**. Ne lancer que si les quatre passent.

## Caveats d'honnêteté (à conserver)
Le dépôt source signale que plusieurs cas emblématiques ont ensuite échoué et
que le chiffre « 86/14 % » est suggestif, non mesuré. À citer tel quel.

## Section d'insertion proposée
Après ERRC, ajouter « Six Paths + 3 cercles de non-clients + Buyer Utility Map +
Séquence stratégique » et les caveats. Attribution : Kim & Mauborgne +
founder-playbook (MIT).
