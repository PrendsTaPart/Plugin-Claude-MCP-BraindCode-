# Patch d'enrichissement — `rapidocrm:mom-test`

> **Cible** : plugin `rapidocrm`, skill `mom-test`.
> **Idées** : Rob Fitzpatrick, *The Mom Test* (2013). **Distillation source** :
> founder-playbook (MIT © 2026 AgentSeal). Reformulation FR, aucun extrait.
> **Ne pas modifier le skill dans ce document.**

Le skill couvre déjà les 3 règles, compliments/fluff, commitment currencies,
customer slicing. Complément mineur :

## 1. Mauvaise donnée type « Idées / Fonctionnalités »
Quand le prospect propose une fonctionnalité, ne pas la noter comme une demande :
**creuser la motivation** derrière (« qu'est-ce que ça vous permettrait ? »).
*Cas Rapido* : lors d'un entretien tracé dans RapidoCRM, distinguer demande réelle
et idée en l'air.

## 2. Filter Test
S'ils n'ont même pas cherché une solution par eux-mêmes (Google, bricolage), le
problème n'est pas assez douloureux pour qu'ils paient.

## Section d'insertion proposée
Compléter la liste des « mauvaises données » avec « Idées/Fonctionnalités » et
ajouter le Filter Test. Attribution : Fitzpatrick + founder-playbook (MIT).
