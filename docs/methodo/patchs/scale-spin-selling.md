# Patch d'enrichissement — `rapido-forge:scale-spin-selling`

> **Cible** : plugin `rapido-forge`, skill `scale-spin-selling`.
> **Idées** : Neil Rackham, *SPIN Selling* (1988). **Distillation source** :
> founder-playbook (MIT © 2026 AgentSeal). Reformulation FR, aucun extrait.
> **Ne pas modifier le skill dans ce document** — proposition d'insertion.

Le skill couvre déjà Situation / Implication / Need-payoff. **Frameworks
manquants** à ajouter :

## 1. Questions PROBLEM (le maillon absent)
Sonder les difficultés, insatisfactions et frustrations actuelles. Objectif :
plusieurs questions Problem par entretien, en ayant **planifié à l'avance** 3
problèmes que l'offre résout. *Cas Rapido* : dans `machine-outbound` /
`scale-bant-qualification`, formuler les Problem à partir des données CRM du
prospect.

## 2. Besoins IMPLICITES → EXPLICITES
Dans les ventes importantes, seuls les **besoins explicites** (le client énonce
lui-même le manque) prédisent l'achat. Le rôle des questions Implication est de
faire passer un problème perçu comme mineur au rang de besoin explicite.

## 3. Les 4 étapes de l'entretien + l'« Advance »
Préliminaires (brefs) → Investigation (là où on gagne) → Démonstration de
capacité (après le besoin, pas avant) → Obtention d'un **engagement**. Juger la
réussite au **comportement** (une commande ou un « Advance » — étape suivante
concrète), pas aux compliments.

## 4. FAB recadré
Un vrai **bénéfice** répond à un besoin **explicite** déjà exprimé ; sinon ce
n'est qu'un « avantage ». Cesser de déballer des fonctionnalités avant le besoin.

## Articulation
Complète `scale-bant-qualification` : SPIN **construit** le « Need » (via
Implication → coût du statu quo) là où BANT ne fait que le cocher.

## Section d'insertion proposée (dans le SKILL.md cible)
Après la partie sur les questions SPIN, ajouter une sous-section
« Problem → Implicite/Explicite → Advance » et un renvoi vers
`scale-bant-qualification`. Attribution : Rackham + founder-playbook (MIT).
