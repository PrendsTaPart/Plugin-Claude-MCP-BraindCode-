# Obviously Awesome — positionnement délibéré

> **Idées** : April Dunford, *Obviously Awesome* (2019). **Distillation source** :
> founder-playbook (MIT © 2026 AgentSeal). Reformulation FR intégrale, aucun
> extrait. Licence : `docs/methodo/100m-leads/NOTICE.md`.

## L'idée centrale
Le **contexte** décide si un produit paraît bon, mauvais ou hors-sujet. Le
positionnement, c'est **choisir délibérément le contexte** dans lequel votre
produit est manifestement le meilleur pour un marché qui y tient.

## Les 5 (+1) composantes (chaîne de dépendance)
1. **Alternatives concurrentes** — ce que le client ferait sans vous (souvent
   « un tableur » ou « rien »), vu de LUI.
2. **Attributs uniques** — ce que vous avez et qu'elles n'ont pas (avec preuve).
3. **Valeur** — le bénéfice que ces attributs permettent (1-4 thèmes).
4. **Caractéristiques du marché cible** — qui s'en soucie le plus.
5. **Catégorie de marché** — le cadre qui rend votre valeur évidente.
6. (optionnel) **Tendance** pertinente — à superposer, sans forcer.
Mauvais ordre = tout s'effondre.

## Les 3 styles de positionnement (le choix clé)
- **Tête-à-tête** : gagner une catégorie existante (coûteux).
- **Gros poisson, petit bassin** : dominer un **sous-segment** (souvent le
  meilleur choix pour une jeune structure).
- **Créer une nouvelle catégorie** : rare, cher, à réserver aux cas solides.

## Distinctions à tenir
**Positionnement** (doc stratégique interne) ≠ **messaging** (traduction client)
≠ **branding** (associations émotionnelles). On positionne d'abord, on rédige
ensuite.

## Exemple Rapido / FoodEatUp
Un logiciel resto qui se repositionne « gros poisson / petit bassin » sur les
**food-trucks** plutôt que « énième logiciel de caisse » — le contexte rend sa
valeur évidente.

## Articulation avec nos skills (pas de doublon)
- `bootcamp-positioning-map` (rapido-forge) = **Perceptual Map** (2 axes) : utile
  pour VISUALISER, mais ne fournit pas la **chaîne des 5 composantes** ni le
  **choix des 3 styles** — c'est l'apport de cette fiche.
- `storybrand-messaging` prend le relais pour **traduire** le positionnement en
  copy ; `icp-generator` fournit les **segments réels** (données CRM) pour l'étape
  « qui s'en soucie le plus ».
- Renvois croisés, déclencheurs non dupliqués (cette fiche = référence méthodo).
