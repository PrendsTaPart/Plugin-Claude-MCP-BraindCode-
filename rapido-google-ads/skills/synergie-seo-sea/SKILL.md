# Synergie SEO / SEA — la boucle qui finance les tests

Le skill différenciant : chaque euro **économisé** en SEA (mot-clé déjà gagné en
organique) **nourrit le budget test** sur les opportunités.

## Étape 0
`reference/garde-fous-sea.md`. Nécessite les données **GSC** (via `rapido-seo` si
installé) et **Google Ads**. Serveur absent = volet sauté **en le disant**.

## Sense
- **GSC** (positions organiques) via `rapido-seo:performance-organique`.
- **Google Ads** (mots-clés payés, coût, conversions).

## Plan (les 3 croisements)
- **Économies** : mots-clés où l'on **paie** alors qu'on **ranke déjà top 3** en
  organique → réduire/couper le paid (le clic organique est gratuit), sauf raison
  (defensive branding). Chiffrer l'économie mensuelle.
- **Opportunités paid** : mots-clés **organiques faibles** (position > 10) à **fort
  volume/intention** → tester en payant le temps que l'organique monte.
- **Cohérence des messages** : title organique vs annonce payante alignés sur la même
  promesse (pas de dissonance).

## Report — une page
Économies chiffrées (à réallouer), opportunités paid priorisées, incohérences de
message — la **boucle** : € économisé SEA → budget test. Actions **manuelles**.

## Garde-fous
Lecture seule ; économies/opportunités **chiffrées et sourcées** (GSC + Google Ads) ;
budget réalloué dans les **plafonds KB** ; calculs via catalogue-kpi.
