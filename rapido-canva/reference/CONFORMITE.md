# Conformité — données envoyées à Canva

Règles applicables à TOUTE génération ou édition de design via le serveur Canva.

## Données personnelles clients : JAMAIS dans un design public

- Un design destiné à être diffusé (post social, flyer, menu, story) ne
  contient JAMAIS de données personnelles de clients : noms de clients
  particuliers, emails, téléphones, historiques d'achat, notes de réservation
  (allergies, VIP…).
- Les données AGRÉGÉES et anonymes sont autorisées (ex. « 4,8/5 sur 320
  avis », « 99 % de livraisons à l'heure »).
- Les supports commerciaux B2B adressés à UN destinataire (proposition,
  présentation de vente) peuvent contenir le nom de l'ENTREPRISE cliente et
  de son contact professionnel — c'est leur objet — mais jamais de données
  d'autres clients, de prix consentis à d'autres, ni de données internes
  (salaires, marges par client).
- En cas de doute sur une donnée : demander à l'utilisateur avant de
  l'envoyer dans une query Canva.

## Traçabilité des chiffres

- Toute génération qui contient des chiffres (prix, stats, preuves) CITE sa
  source dans la conversation : outil MCP (ex. « prix de list_dishes »),
  fichier KB (ex. « preuve de propositions-valeur.md »), ou l'utilisateur.
- Aucun chiffre inventé pour « faire joli » dans un design : un chiffre sans
  source ne part pas chez Canva.

## Rappels généraux

- Les exports produisent des URLs de téléchargement : les transmettre à
  l'utilisateur, ne pas les rediffuser ailleurs sans demande.
- Le contenu envoyé à Canva transite par un service tiers : ne jamais y mettre
  de secrets (tokens, identifiants, données bancaires).
