# Netlinking (DataForSEO Backlinks)

Cartographie le profil de liens et trouve les opportunités. **Analyse**, puis
délègue l'outreach.

## Étape 0
`reference/garde-fous-seo.md` (coûts backlinks) ; historique
`./rapido-kb/seo/netlinking.md` ; concurrents `./rapido-kb/seo/seo-cibles.md`.

## Sense (DataForSEO Backlinks — famille `backlinks`)
- **Résumé du domaine** (referring domains, backlinks totaux, autorité), **new/lost**
  (30 j), profil d'ancres, top pages liées.
- **Comparaison 2-3 concurrents** : referring domains qu'ils ont et pas nous.
- **Coût** : backlinks bulk = **facturé** → **annoncé avant** (garde-couts-seo) ;
  volume récurrent → n8n.

## Plan (gap analysis)
- **Opportunités de liens** = referring domains des concurrents absents de notre
  profil, classés par autorité et pertinence. Ancres visées cohérentes (pas de
  sur-optimisation).

## Act (outreach délégué)
- Le **plan d'outreach** (qui contacter, angle, ancre) est **délégué à
  `rapidocrm:draft-outreach`** (brouillons confirmés — jamais d'envoi d'office).
- **Historique** consigné dans `./rapido-kb/seo/netlinking.md` (cible, statut, ancre).

## Report — une page
Profil de référence (total / new / lost, daté), top opportunités priorisées, plan
d'outreach préparé, coût backlinks consommé.

## Garde-fous
Coût DataForSEO annoncé ; ancres **naturelles** (pas de spam) ; outreach **confirmé** ;
new/lost **datés** ; serveur absent = le dire.
