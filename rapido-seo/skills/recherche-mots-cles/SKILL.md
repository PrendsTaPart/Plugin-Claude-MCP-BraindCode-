# Recherche de mots-clés (DataForSEO Labs/Keywords)

Trouve les requêtes qui valent le coup et les mappe aux pages. **Search classique**
(organique) — pas l'optimisation pour moteurs génératifs (voir Anti-collision).

## Étape 0
`reference/garde-fous-seo.md` (coûts) ; personas `./rapido-kb/marketing/icp.md` ;
cibles existantes `./rapido-kb/seo/seo-cibles.md`.

## Sense (DataForSEO)
- **Volumes & CPC** (famille `keywords_data`), **difficulté & mots-clés rankés**
  (famille `dataforseo_labs`), **questions / longue traîne** (suggestions, related).
- Partir des **thèmes du client** et des **personas ICP** (besoins, vocabulaire) —
  pas une liste hors-sol.
- **Coût** : requêtes Labs/Keywords sur listes = **annoncées avant** (garde-couts-seo).

## Plan
- Pour chaque mot-clé : **intention** (informationnelle / commerciale /
  transactionnelle / navigationnelle), **difficulté vs autorité du site**, **page
  cible** (existante ou à créer), **priorité** (volume × intention × faisabilité,
  formule affichée — seuils `./rapido-kb/seo/`).

## Livrable
- **Mapping mot-clé → page → intention → priorité** écrit dans
  `./rapido-kb/seo/seo-cibles.md` (proposé, confirmé). Clusters par page pilier.

## Anti-collision
- **`rapido-marketing:geo-optimization`** = optimisation pour **moteurs GÉNÉRATIFS**
  (réponses LLM, GEO). **Moi = search classique** (Google, volumes, SERP). Règle
  miroir documentée dans les deux SKILL.md ; un besoin « être cité par ChatGPT » →
  geo-optimization ; « me positionner sur Google » → ici.

## Garde-fous
Coût DataForSEO annoncé ; intention **sourcée** ; mapping écrit **sur confirmation** ;
données réelles ; serveur absent = le dire.
