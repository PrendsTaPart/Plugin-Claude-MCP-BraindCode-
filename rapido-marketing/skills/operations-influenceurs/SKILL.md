# Opérations influenceurs — sourcer, briefer, contractualiser, tracker, ROI

Opère les collaborations **une par une**, de bout en bout, sur données réelles.

## Étape 0 — Pont forge + frontières
- Livrable forge `scale-influencer-marketing` (stratégie) comme cadre ; absent → défauts en le disant.
- **Frontières** : `scale-influencer-marketing` = **concevoir la stratégie** (exercice) ;
  `lead-getters-systeme` = **choisir le TYPE** de programme d'apport ; **moi = OPÉRER**
  les collaborations. Documentées des trois côtés.

## (a) SOURCING
- **Grille de sélection** depuis la KB (`./rapido-kb/marketing/`) : audience vs personas
  ICP (`./rapido-kb/marketing/icp.md`), taux d'engagement estimé, cohérence marque.
- **Recherche** (web) pour identifier des candidats par niche (restauration,
  entrepreneuriat, PME) ; enregistrer les retenus comme **entreprises CRM taguées
  « influenceur »** (`create_entreprise`) — après confirmation.

## (b) BRIEF
- **Brief de campagne conforme charte** — délégué à `rapidocms:contenu-conforme-marque`.
- **Mentions légales de transparence** (« collaboration commerciale » / partenariat
  rémunéré) **OBLIGATOIRES dans tout brief** (conformité).

## (c) CONTRAT
- Délégué à `rapidocrm:contrats-clients` (template **contrat influenceur** à créer dans
  la KB) — jamais de contrat envoyé sans confirmation.

## (d) TRACKING
- **Un code promo OU une UTM par influenceur** (créés côté CRM/CMS) ; leads et ventes
  **attribués** via les soumissions de formulaires et les factures.

## (e) ROI
- **Coût de la collaboration vs CA attribué** — calcul via `rapido-startup:catalogue-kpi`
  (jamais de tête). Classement des influenceurs par **ROI** dans
  `./rapido-kb/marketing/influenceurs.md`.

## Garde-fous
Sélection **sourcée** (grille KB, jamais un influenceur « au feeling ») ; **mentions
légales obligatoires** ; contrat/tracking **confirmés** ; ROI via catalogue-kpi ;
consentement RGPD ; aucun chiffre d'engagement inventé (estimé = dit comme tel).
