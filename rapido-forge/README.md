# rapido-forge — StartupsForge / PrendsTaPart

180 exercices d'incubateur, mis aux conventions du marketplace rapido, en
3 parcours :

- **Bootcamp 5 jours** (46 skills `bootcamp-*`) — de l'idée brute au plan
  de lancement, timeboxé, animé par `mentor-bootcamp`.
- **Roadmap idéation** (72 skills `ideation-*`) — valider, construire,
  lancer, avec `mentor-ideation`.
- **Roadmap scale** (62 skills `scale-*`) — mesurer, vendre, acquérir,
  financer, avec `mentor-scale`.

`directeur-programme` diagnostique la maturité du projet et route vers le
bon parcours. Le journal de parcours vit dans
`./rapido-kb/startup/forge/parcours.md`, les livrables dans
`./rapido-kb/startup/forge/<parcours>/`.

## Conventions appliquées (vs source StartupsForge)

- Descriptions frontmatter normalisées (« Utiliser quand… », parcours
  identifié) pour un déclenchement fiable.
- Étape 0 systématique : KB d'abord, chiffres via MCP, hypothèses marquées.
- Livrables datés dans la KB du client, jamais dans le plugin.
- MCP corrects par thème (Stripe lecture + script pour la finance, Meta
  ads en PAUSED, Lovable, n8n, Canva, HeyGen…).
- Les 46 skills bootcamp livrés vides ont reçu une méthode en 5-6 étapes.
- Renvois « Voir aussi » vers les skills plus riches du marketplace
  (~90 correspondances) — la forge prépare, les plugins métier exécutent.

## Dépendances conseillées

rapido-suite (KB), rapido-startup (KPI/BP), rapidocrm, rapidocms,
rapido-lovable, rapido-meta-ads, rapido-n8n, rapido-canva, rapidorh.
