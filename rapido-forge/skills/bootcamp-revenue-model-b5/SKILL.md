# Modèle de Revenus

**Bootcamp 5 jours — Jour 3**  
**Catégorie** : Stratégie  
**Framework** : Revenue Model Framework  
**Durée** : 40 min

## Objectif

Définis tes sources de revenus et pricing

## Étape 0 — Contexte (obligatoire)

Charger `./rapido-kb/` — surtout `./rapido-kb/startup/` (vision, persona,
offre, hypothèses construits par `dossier-startup-360` et
`interview-business-plan`). La KB PRIME : ne jamais redemander une
information déjà validée, la reformuler pour confirmation. Les chiffres
réels viennent des MCP — jamais de mémoire ; tout chiffre sans source datée
porte la mention « hypothèse fondateur, confiance faible ».

## Étapes

1. **Lister les modèles candidats** — abonnement, usage, commission, licence, service, freemium, mix
2. **Croiser avec le persona** — comment PAIE-t-il déjà pour résoudre ce problème ? S'aligner sur son habitude
3. **Choisir 1 modèle principal** — et une hypothèse de prix ancrée sur la valeur créée (pas sur les coûts)
4. **Unit economics grossiers** — prix − coût de service − coût d'acquisition estimé : positif à terme ?
5. **Plan de test** — comment valider le consentement à payer en 2 semaines (précommande, LOI, devis signé)

## Données & serveurs MCP

- **RapidoCRM** (`rapidocrm`) — données réelles, lecture d'abord
