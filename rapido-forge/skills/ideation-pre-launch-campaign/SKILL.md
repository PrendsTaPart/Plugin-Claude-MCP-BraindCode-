# Pre Launch Campaign

**Catégorie** : Idéation  
**Durée** : 45-60 min

## Pourquoi

Une campagne de pré-lancement construit l'anticipation et te donne une liste de prospects chauds pour le Jour J.

## Objectif

Créer une campagne email de pré-lancement.

## Livrable attendu

Séquence de 3-5 emails avant lancement

## Étape 0 — Contexte (obligatoire)

Charger `./rapido-kb/` — surtout `./rapido-kb/startup/` (vision, persona,
offre, hypothèses construits par `dossier-startup-360` et
`interview-business-plan`). La KB PRIME : ne jamais redemander une
information déjà validée, la reformuler pour confirmation. Les chiffres
réels viennent des MCP — jamais de mémoire ; tout chiffre sans source datée
porte la mention « hypothèse fondateur, confiance faible ».

## Étapes

1. **Créer la landing page waitlist** — Teaser + formulaire
2. **Email J-14 : Teaser** — Annonce quelque chose de gros
3. **Email J-7 : Révélation** — Montre ce que c'est
4. **Email J-1 : Countdown** — Dernière chance early access
5. **Email Jour J : Launch** — C'est parti !

## Pro tips

- Offre un avantage exclusif aux inscrits waitlist
- Ajoute un système de parrainage
- Crée l'urgence avec un compteur

## Erreurs fréquentes

- Trop d'emails qui spamment
- Pas de valeur ajoutée dans les emails
- Landing page waitlist mal optimisée

## Données & serveurs MCP

- **RapidoCMS** (`rapidocms`) — données réelles, lecture d'abord

## Livrable — toujours dans la KB du client

Écrire le livrable daté dans
`./rapido-kb/startup/forge/ideation/ideation-pre-launch-campaign.md` (le dossier du client,
dans SON répertoire de travail — jamais dans le plugin). Si des chiffres
clés en sortent, proposer de mettre à jour
`./rapido-kb/startup/business-plan/hypotheses.md` (source + date).
