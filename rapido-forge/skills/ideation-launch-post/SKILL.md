# Launch Post

**Catégorie** : Idéation  
**Durée** : 30-40 min

## Pourquoi

Le post de lancement peut générer des milliers de vues et tes premiers clients. Un lancement bien orchestré crée du momentum.

## Objectif

Rédiger le post de lancement parfait pour tes réseaux.

## Livrable attendu

Post de lancement formaté pour LinkedIn, Twitter et Product Hunt

## Étape 0 — Contexte (obligatoire)

Charger `./rapido-kb/` — surtout `./rapido-kb/startup/` (vision, persona,
offre, hypothèses construits par `dossier-startup-360` et
`interview-business-plan`). La KB PRIME : ne jamais redemander une
information déjà validée, la reformuler pour confirmation. Les chiffres
réels viennent des MCP — jamais de mémoire ; tout chiffre sans source datée
porte la mention « hypothèse fondateur, confiance faible ».

## Étapes

1. **Écrire le hook** — Annonce + pourquoi c'est important
2. **Décrire la solution** — Ce que tu lances, pour qui
3. **Ajouter la preuve sociale** — Résultats beta, témoignages
4. **Inclure le CTA** — Lien, offre de lancement
5. **Adapter pour chaque plateforme** — Format et ton différents

## Pro tips

- Préviens ta communauté avant le lancement
- Demande explicitement les upvotes/likes
- Réponds à TOUS les commentaires le jour J

## Erreurs fréquentes

- Lancer sans communauté préalable
- Post trop corporate et froid
- Pas de suivi après le lancement

## Données & serveurs MCP

- **RapidoCMS** (`rapidocms`) — données réelles, lecture d'abord

## Livrable — toujours dans la KB du client

Écrire le livrable daté dans
`./rapido-kb/startup/forge/ideation/ideation-launch-post.md` (le dossier du client,
dans SON répertoire de travail — jamais dans le plugin). Si des chiffres
clés en sortent, proposer de mettre à jour
`./rapido-kb/startup/business-plan/hypotheses.md` (source + date).
