# Kpi Dashboard

**Catégorie** : Idéation  
**Durée** : 45-60 min

## Pourquoi

Un dashboard KPI permet de prendre des décisions data-driven en un coup d'œil. C'est le cockpit de ta startup.

## Objectif

Créer un dashboard avec les KPIs essentiels.

## Livrable attendu

Dashboard live avec 5-7 KPIs

## Étape 0 — Contexte (obligatoire)

Charger `./rapido-kb/` — surtout `./rapido-kb/startup/` (vision, persona,
offre, hypothèses construits par `dossier-startup-360` et
`interview-business-plan`). La KB PRIME : ne jamais redemander une
information déjà validée, la reformuler pour confirmation. Les chiffres
réels viennent des MCP — jamais de mémoire ; tout chiffre sans source datée
porte la mention « hypothèse fondateur, confiance faible ».

## Étapes

1. **Lister les KPIs** — NSM + 4-6 input metrics
2. **Choisir l'outil** — Notion, Sheets, Metabase
3. **Connecter les sources** — Analytics, CRM, facturation
4. **Designer le dashboard** — Clair, scannable, actionnable
5. **Automatiser les updates** — Refresh quotidien ou temps réel

## Pro tips

- Max 7 métriques sur un seul écran
- Ajoute des comparaisons période précédente
- Inclus les trends, pas juste les chiffres

## Erreurs fréquentes

- Trop de métriques qui noient l'essentiel
- Données non à jour
- Pas accessible à toute l'équipe

## Données & serveurs MCP

- **RapidoCRM** (`rapidocrm`) — données réelles, lecture d'abord

## Livrable — toujours dans la KB du client

Écrire le livrable daté dans
`./rapido-kb/startup/forge/ideation/ideation-kpi-dashboard.md` (le dossier du client,
dans SON répertoire de travail — jamais dans le plugin). Si des chiffres
clés en sortent, proposer de mettre à jour
`./rapido-kb/startup/business-plan/hypotheses.md` (source + date).
