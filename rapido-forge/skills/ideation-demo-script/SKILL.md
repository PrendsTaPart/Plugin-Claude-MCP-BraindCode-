# Demo Script

**Catégorie** : Idéation  
**Durée** : 45-60 min

## Pourquoi

Une démo produit bien filmée peut remplacer des dizaines de calls commerciaux. C'est ton meilleur vendeur 24/7.

## Objectif

Enregistrer une démo produit professionnelle.

## Livrable attendu

Vidéo démo de 2-5 minutes montrant le produit en action

## Étape 0 — Contexte (obligatoire)

Charger `./rapido-kb/` — surtout `./rapido-kb/startup/` (vision, persona,
offre, hypothèses construits par `dossier-startup-360` et
`interview-business-plan`). La KB PRIME : ne jamais redemander une
information déjà validée, la reformuler pour confirmation. Les chiffres
réels viennent des MCP — jamais de mémoire ; tout chiffre sans source datée
porte la mention « hypothèse fondateur, confiance faible ».

## Étapes

1. **Scripter le parcours** — Scénario utilisateur réaliste
2. **Préparer l'environnement** — Données de démo, bureau propre
3. **Enregistrer avec Tella** — Screen + caméra en simultané
4. **Éditer les moments clés** — Couper les temps morts, accélérer
5. **Ajouter les callouts** — Flèches, zooms, textes explicatifs

## Pro tips

- Commence par le 'aha moment' du produit
- Montre la valeur en < 30 secondes
- Termine par un CTA clair

## Erreurs fréquentes

- Démo trop longue et détaillée
- Commencer par les fonctionnalités ennuyeuses
- Pas de contexte sur le problème résolu

## Données & serveurs MCP

- **RapidoCRM** (`rapidocrm`) — données réelles, lecture d'abord

## Livrable — toujours dans la KB du client

Écrire le livrable daté dans
`./rapido-kb/startup/forge/ideation/ideation-demo-script.md` (le dossier du client,
dans SON répertoire de travail — jamais dans le plugin). Si des chiffres
clés en sortent, proposer de mettre à jour
`./rapido-kb/startup/business-plan/hypotheses.md` (source + date).
