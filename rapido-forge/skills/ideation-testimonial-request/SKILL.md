# Testimonial Request

**Catégorie** : Idéation  
**Durée** : 25-35 min

## Pourquoi

92% des consommateurs lisent les avis avant d'acheter. Les témoignages sont la preuve sociale la plus puissante pour convertir les sceptiques en clients.

## Objectif

Créer des témoignages crédibles pour renforcer la confiance.

## Livrable attendu

3-5 témoignages formatés avec photo, nom, titre et citation

## Étape 0 — Contexte (obligatoire)

Charger `./rapido-kb/` — surtout `./rapido-kb/startup/` (vision, persona,
offre, hypothèses construits par `dossier-startup-360` et
`interview-business-plan`). La KB PRIME : ne jamais redemander une
information déjà validée, la reformuler pour confirmation. Les chiffres
réels viennent des MCP — jamais de mémoire ; tout chiffre sans source datée
porte la mention « hypothèse fondateur, confiance faible ».

## Étapes

1. **Identifier les clients idéaux** — Ceux qui ont eu les meilleurs résultats
2. **Créer l'email de demande** — Questions guidées pour obtenir des réponses exploitables
   > Prompt: Rédige un email pour demander un témoignage à un client satisfait, avec 3 questions guidées
3. **Structurer le témoignage** — Situation avant → Solution utilisée → Résultats après
4. **Formater pour le web** — Photo, prénom, titre, entreprise, citation
5. **Générer si pas de clients** — Témoignages fictifs basés sur tes personas

## Pro tips

- Les témoignages avec chiffres concrets sont 2x plus efficaces
- Inclus des témoignages vidéo si possible
- Varie les profils pour toucher différents segments

## Erreurs fréquentes

- Témoignages trop génériques ('Super produit!')
- Pas de photo ou nom réel (réduit la crédibilité)
- Tous les témoignages disent la même chose

## Données & serveurs MCP

- **RapidoCRM** (`rapidocrm`) — données réelles, lecture d'abord

## Livrable — toujours dans la KB du client

Écrire le livrable daté dans
`./rapido-kb/startup/forge/ideation/ideation-testimonial-request.md` (le dossier du client,
dans SON répertoire de travail — jamais dans le plugin). Si des chiffres
clés en sortent, proposer de mettre à jour
`./rapido-kb/startup/business-plan/hypotheses.md` (source + date).
