# North Star Metric

**Catégorie** : Idéation  
**Durée** : 30-45 min

## Pourquoi

La North Star Metric aligne toute l'équipe sur un objectif commun. C'est le chiffre qui résume le succès de ta startup.

## Objectif

Définir et documenter ta North Star Metric.

## Livrable attendu

NSM + input metrics documentés

## Étape 0 — Contexte (obligatoire)

Charger `./rapido-kb/` — surtout `./rapido-kb/startup/` (vision, persona,
offre, hypothèses construits par `dossier-startup-360` et
`interview-business-plan`). La KB PRIME : ne jamais redemander une
information déjà validée, la reformuler pour confirmation. Les chiffres
réels viennent des MCP — jamais de mémoire ; tout chiffre sans source datée
porte la mention « hypothèse fondateur, confiance faible ».

## Étapes

1. **Identifier la valeur livrée** — Quel moment 'aha' pour l'utilisateur ?
2. **Choisir la métrique** — Mesurable, actionnable, corrélée au revenu
3. **Définir les input metrics** — 4-5 leviers qui influencent la NSM
4. **Fixer des targets** — Objectifs par période
5. **Communiquer à l'équipe** — Dashboard visible, rituels

## Pro tips

- La NSM doit refléter la valeur utilisateur, pas juste le revenu
- Exemples : DAU pour les apps, Weekly Active Customers pour SaaS
- Revois la NSM tous les trimestres

## Erreurs fréquentes

- NSM trop vague ou trop complexe
- Choisir une vanity metric
- Ne pas la rendre visible à tous

## Données & serveurs MCP

- **RapidoCRM** (`rapidocrm`) — données réelles, lecture d'abord

## Livrable — toujours dans la KB du client

Écrire le livrable daté dans
`./rapido-kb/startup/forge/ideation/ideation-north-star-metric.md` (le dossier du client,
dans SON répertoire de travail — jamais dans le plugin). Si des chiffres
clés en sortent, proposer de mettre à jour
`./rapido-kb/startup/business-plan/hypotheses.md` (source + date).
