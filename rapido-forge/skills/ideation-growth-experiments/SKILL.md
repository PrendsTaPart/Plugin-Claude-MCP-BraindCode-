# Growth Experiments

**Catégorie** : Idéation  
**Durée** : 45-60 min

## Pourquoi

La croissance vient de l'expérimentation systématique. Chaque test t'apprend quelque chose, même les échecs.

## Objectif

Planifier 3 expériences de croissance.

## Livrable attendu

3 expériences documentées avec hypothèses

## Étape 0 — Contexte (obligatoire)

Charger `./rapido-kb/` — surtout `./rapido-kb/startup/` (vision, persona,
offre, hypothèses construits par `dossier-startup-360` et
`interview-business-plan`). La KB PRIME : ne jamais redemander une
information déjà validée, la reformuler pour confirmation. Les chiffres
réels viennent des MCP — jamais de mémoire ; tout chiffre sans source datée
porte la mention « hypothèse fondateur, confiance faible ».

## Étapes

1. **Identifier les leviers** — Acquisition, Activation, Retention
2. **Former les hypothèses** — Si X alors Y (+Z%)
3. **Prioriser avec ICE** — Impact × Confidence × Ease
4. **Designer les tests** — Méthodo, durée, success metrics
5. **Documenter les résultats** — Learnings pour le futur

## Pro tips

- Une expérience = une variable
- Documente même les échecs
- Célèbre l'apprentissage, pas juste les wins

## Erreurs fréquentes

- Tester trop de choses à la fois
- Pas assez de volume pour des résultats significatifs
- Arrêter trop tôt

## Données & serveurs MCP

- **RapidoCRM** (`rapidocrm`) — données réelles, lecture d'abord

## Livrable — toujours dans la KB du client

Écrire le livrable daté dans
`./rapido-kb/startup/forge/ideation/ideation-growth-experiments.md` (le dossier du client,
dans SON répertoire de travail — jamais dans le plugin). Si des chiffres
clés en sortent, proposer de mettre à jour
`./rapido-kb/startup/business-plan/hypotheses.md` (source + date).
