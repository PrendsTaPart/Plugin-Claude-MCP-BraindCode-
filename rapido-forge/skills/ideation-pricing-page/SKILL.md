# Pricing Page

**Catégorie** : Idéation  
**Durée** : 35-45 min

## Pourquoi

La page Pricing est décisive pour la conversion. 70% des visiteurs quittent si le pricing n'est pas clair. Une bonne page pricing augmente les conversions de 30%.

## Objectif

Présenter tes tarifs de façon claire et convaincante.

## Livrable attendu

Page Pricing avec 2-3 plans, features comparatives et FAQ

## Étape 0 — Contexte (obligatoire)

Charger `./rapido-kb/` — surtout `./rapido-kb/startup/` (vision, persona,
offre, hypothèses construits par `dossier-startup-360` et
`interview-business-plan`). La KB PRIME : ne jamais redemander une
information déjà validée, la reformuler pour confirmation. Les chiffres
réels viennent des MCP — jamais de mémoire ; tout chiffre sans source datée
porte la mention « hypothèse fondateur, confiance faible ».

## Étapes

1. **Définir les plans** — 2-4 plans avec différenciation claire
2. **Nommer les plans** — Noms évocateurs (Starter, Pro, Enterprise)
3. **Lister les features** — Tableau comparatif visuel
4. **Ajouter le plan recommandé** — Mets en avant le plan que tu veux vendre
5. **Rédiger la FAQ pricing** — Anticipe les objections

## Pro tips

- Utilise l'effet d'ancrage (montre le plus cher d'abord)
- Mets en avant le plan 'recommandé' visuellement
- Inclus une garantie ou période d'essai

## Erreurs fréquentes

- Trop d'options créent la paralysie
- Différenciation pas claire entre les plans
- Pas de CTA sur chaque plan

## Données & serveurs MCP

- **RapidoCMS** (`rapidocms`) — données réelles, lecture d'abord
- **Lovable** (`lovable`, via le plugin `rapido-lovable`) pour la construction réelle
