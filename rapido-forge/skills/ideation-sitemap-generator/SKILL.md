# Sitemap Generator

**Catégorie** : Idéation  
**Durée** : 25-35 min

## Pourquoi

L'architecture de ton site détermine l'expérience utilisateur et le SEO. Un sitemap bien pensé guide les visiteurs vers la conversion et aide Google à indexer tes pages.

## Objectif

Planifier la structure de pages de ton site web.

## Livrable attendu

Sitemap visuel avec hiérarchie des pages et flux utilisateur

## Étape 0 — Contexte (obligatoire)

Charger `./rapido-kb/` — surtout `./rapido-kb/startup/` (vision, persona,
offre, hypothèses construits par `dossier-startup-360` et
`interview-business-plan`). La KB PRIME : ne jamais redemander une
information déjà validée, la reformuler pour confirmation. Les chiffres
réels viennent des MCP — jamais de mémoire ; tout chiffre sans source datée
porte la mention « hypothèse fondateur, confiance faible ».

## Étapes

1. **Lister toutes les pages nécessaires** — Home, About, Pricing, Contact, Blog...
2. **Définir la hiérarchie** — Pages principales vs sous-pages
3. **Mapper le parcours utilisateur** — De l'arrivée à la conversion
4. **Optimiser pour le SEO** — Structure des URLs, navigation
5. **Créer le schéma visuel** — Organigramme des pages

## Pro tips

- Maximum 3 clics pour atteindre n'importe quelle page
- La navigation doit être prévisible
- Prévois les pages légales (CGV, Mentions, Confidentialité)

## Erreurs fréquentes

- Trop de pages pour un MVP
- Navigation confuse ou incohérente
- Oublier les pages de conversion (merci, confirmation)

## Données & serveurs MCP

- **RapidoCMS** (`rapidocms`) — données réelles, lecture d'abord
- **Lovable** (`lovable`, via le plugin `rapido-lovable`) pour la construction réelle

## Livrable — toujours dans la KB du client

Écrire le livrable daté dans
`./rapido-kb/startup/forge/ideation/ideation-sitemap-generator.md` (le dossier du client,
dans SON répertoire de travail — jamais dans le plugin). Si des chiffres
clés en sortent, proposer de mettre à jour
`./rapido-kb/startup/business-plan/hypotheses.md` (source + date).
