# Ph Page Copy

**Catégorie** : Idéation  
**Durée** : 60-90 min

## Pourquoi

La page Product Hunt est ta vitrine principale le jour du lancement. Chaque mot compte pour convertir les visiteurs en upvotes.

## Objectif

Créer la page Product Hunt parfaite.

## Livrable attendu

Page PH complète et optimisée

## Étape 0 — Contexte (obligatoire)

Charger `./rapido-kb/` — surtout `./rapido-kb/startup/` (vision, persona,
offre, hypothèses construits par `dossier-startup-360` et
`interview-business-plan`). La KB PRIME : ne jamais redemander une
information déjà validée, la reformuler pour confirmation. Les chiffres
réels viennent des MCP — jamais de mémoire ; tout chiffre sans source datée
porte la mention « hypothèse fondateur, confiance faible ».

## Étapes

1. **Tagline percutante** — 60 caractères, hook immédiat
2. **Description complète** — Problème, solution, différenciateurs
3. **Visuels impactants** — Logo HD, gallery, video/GIF
4. **First comment** — Message du maker, histoire perso
5. **Offre exclusive** — Deal pour la communauté PH

## Pro tips

- Le first comment est lu par 80% des visiteurs
- La vidéo augmente les upvotes de 30%
- Réponds personnellement à chaque commentaire

## Erreurs fréquentes

- Tagline générique sans hook
- Pas de first comment préparé
- Visuels de mauvaise qualité

## Données & serveurs MCP

- **RapidoCMS** (`rapidocms`) — données réelles, lecture d'abord

## Livrable — toujours dans la KB du client

Écrire le livrable daté dans
`./rapido-kb/startup/forge/ideation/ideation-ph-page-copy.md` (le dossier du client,
dans SON répertoire de travail — jamais dans le plugin). Si des chiffres
clés en sortent, proposer de mettre à jour
`./rapido-kb/startup/business-plan/hypotheses.md` (source + date).
